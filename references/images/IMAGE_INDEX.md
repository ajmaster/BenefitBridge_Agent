## Suggested UX Narrative

1. Start with `18-desktop-prep-ready-homepage.png` or `16-mobile-onboarding-benefit-cards.png`.
2. Intake with `12-desktop-household-snapshot-intake.png` or `03-mobile-chat-intake-flow.png`.
3. Review benefit matches with `13-desktop-benefit-paths-dashboard.png`.
4. Drill into one benefit using `14-desktop-calfresh-path-detail.png`.
5. Find handoffs using `06-mobile-map-resource-finder.png` or `17-mobile-resource-map-flow.png`.
6. Export the final packet using `15-desktop-prep-packet-editor-export.png`, `08-printable-family-prep-packet.png`, or `19-printable-desktop-prep-workspace.png`.

## Implementation Caveats

- Do not copy any text in the images as policy language. Use the docs source pack for exact wording.
- Avoid eligibility guarantees, benefit amounts, or real-time availability claims even if a mockup visually implies confidence.
- Any resource-locator UI must include call-before-going behavior for food, shelter, legal aid, WIC, and unstable office-hour contexts.
- The final app should prioritize dense, calm, task-oriented UI over a marketing landing page.
